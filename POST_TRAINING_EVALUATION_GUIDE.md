## 1. What can be generated from `metrics.jsonl`

These graphs do not require loading the model.

The script can generate:

| Graph | Meaning |
|---|---|
| Loss curves | Generative loss and recognition/metric loss over epochs. |
| Rank curves | Rank-1, Rank-5, and best Rank-1 over training. |
| Verification curves | Verification AUC and verification accuracy over training. |
| Distance curves | Same-person distance, different-person distance, and distance gap. |
| Learning-rate curve | Shows cosine LR decay across training. |
| Step-count curve | Shows how many generative vs recognition steps happened per epoch. |

Command:

```bash
python tools/post_training_analysis.py --run-dir runs/fusion_rank1_002 --skip-model-eval
```

Output:

```text
runs/fusion_rank1_002/post_training_analysis/
```

Because the current laptop Python environment does not have `matplotlib`, the script generated dependency-free `.svg` graphs. These can be opened in a browser or inserted into documents/presentations.

Generated files:

```text

01_losses.svg
02_retrieval_rank.svg
03_verification.svg
04_distances.svg
05_learning_rate.svg
06_training_steps.svg
POST_TRAINING_REPORT.md
training_summary.json
```

## 3. What can be generated by loading the `.pt` model

If PyTorch is available locally, the same script can load the trained model and evaluate it on the local test dataset.

This can generate:

| Output | Meaning |
|---|---|
| Local Rank-1 / Rank-3 / Rank-5 / Rank-10 | Retrieval accuracy at multiple ranks. |
| Condition-wise Rank metrics | Performance split by normal/bag/clothing conditions. |
| Verification AUC | Pairwise same/different separation quality. |
| EER estimate | Equal-error-rate estimate for verification. |
| Best-threshold precision/recall/F1 | Pairwise verification threshold behavior. |
| Distance histogram | Distribution of same-person vs different-person distances. |
| ROC curve | Verification tradeoff curve. |
| CMC curve | Retrieval accuracy across Rank-k. |
| Embedding PCA scatter | 2D visualization of embedding clusters. |
| CSV probe rows | Per-probe rank and top-1 match details. |
| NPZ embeddings | Saved embeddings and pair scores for later analysis. |

## 4. Local environment requirement for `.pt` evaluation

The default local Python reported:

```text
ModuleNotFoundError: No module named 'torch'
ModuleNotFoundError: No module named 'matplotlib'
```

So graph-only mode works with the default Python, but local model evaluation needs a PyTorch environment.

The conda environment named `ML` has been checked and contains:

```text
torch 2.9.1
matplotlib 3.10.8
cuda false
mps built true
mps available false in the current shell
```

So use the `ML` environment for local evaluation. The script now automatically uses:

```text
CUDA if available -> MPS if available -> CPU fallback
```

On the current machine/session, PyTorch reported MPS as not available, so the completed local evaluation used CPU.

Recommended setup:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

If installing full PyTorch locally is heavy, run the model-evaluation command inside Modal instead, or use a machine that already has PyTorch.

## 5.1 Important local dataset requirement

The local evaluation requires the full paired silhouette dataset.

During the first local evaluation attempt, the local folders contained:

```text
datasets/CASIA_B_Hamilton_Skeleton: 124 subjects
datasets/GaitDatasetB-silh:          subject 001 extracted + many subject .tar.gz files
```

Initially, the preprocessing reader only saw the extracted `001/` folder and ignored the nested `.tar.gz` files. That created:

```text
paired sequences: 24
subjects:         1
missing_pairs:    2940
```

This was fixed by updating the preprocessing reader so it reads both direct image folders and nested `.tar.gz` archives inside a dataset directory.

After the fix, the local reader found:

```text
silhouette sequences: 13593
silhouette subjects:  124
paired V6 sequences:  2964
missing_pairs:        0
```

So manual unzipping of every subject archive is not required.

To fix this, download or extract the full silhouette dataset so this folder contains all matching subjects:

```text
datasets/GaitDatasetB-silh/
  001/
  002/
  003/
  ...
  124/
```

If the full silhouette dataset is missing entirely, download it locally with the Modal CLI. The command should be similar to:

```bash
modal volume get gait-datasets-store /GaitDatasetB-silh.zip GaitDatasetB-silh.zip
```

Then either unzip it, or keep subject `.tar.gz` archives inside:

```text
datasets/GaitDatasetB-silh/
```

After replacing the dataset, rerun local evaluation with:

```bash
--force-preprocess
```

or use a new cache path:

```bash
--cache-dir runs/fusion_rank1_002/local_processed_cache_full
```

## 7. Which checkpoint should be reported?

Recommended reporting:

| Checkpoint | Use for |
|---|---|
| `best_rank1_model.pt` | Best retrieval result. |
| `best_model.pt` | Best verification-AUC result. |
| `latest.pt` | Resume/debug only; not usually the best report checkpoint. |

For Opshora's thesis, the most important checkpoint is probably `best_model.pt` because her core claim is blind verification / contrastive distance separation.

But Rank-1 is also important for presentation, so also report the best Rank-1 value from `best_rank1_model.pt` or from `metrics.jsonl`.

## 8. Important reporting wording

Because V6 uses:

```json
"eval_gallery_per_subject": 3
```

write:

```text
3-gallery Rank-1 = 0.6199
```

Do not present it as single-gallery Rank-1.

For verification:

```text
Verification AUC = 0.9078
```

This is a strong thesis metric because it directly measures whether same-person pairs are closer than different-person pairs.
